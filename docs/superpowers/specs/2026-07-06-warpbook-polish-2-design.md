# warpbook P2 — полишинг-спринт №2 (дизайн)

Дата: 2026-07-06. Позиция: между M7b «Исполняемые ячейки» (SHIPPED) и M10 «Автопрофиль» (`docs/ROADMAP.md`). Первый полишинг-спринт был частью M5 (2026-06-26); с тех пор бэклог (`docs/BACKLOG.md`) накопил ~35 открытых пунктов. P2 закрывает их одним скоупом + чинит мобильную вёрстку (прямой запрос юзера, 7 скриншотов с телефона 2026-07-06).

## Контекст

warpbook — десктоп-инструмент; мобилка вне скоупа v1, но демку показывают с телефона. Диагноз по скриншотам:

- **Портрет.** Брейкпоинт `@media (max-width: 720px)` уже есть (рейл встаёт колонкой над воркспейсом), но ломают две вещи:
  1. `.topbar` не переносится — его min-content (~650px: лого + режимы + `?` + engine-tag + local + reset) распирает страницу шире вьюпорта → горизонтальный скролл всей страницы, мёртвое чёрное поле справа (бэклог M-5, фронт-ревью 2026-07-05).
  2. `.shell { height: 100vh }` с внутренними скролл-зонами несовместим с колоночным режимом: рейл (`flex: none`, интринсик-высота со схемой) выталкивает контент за пределы 100vh-клетки, статуслайн повисает посреди контента, ниже — фон страницы.
- **Альбом.** Вёрстка цела (~915px CSS — уже десктопная), но всё крупное и тесно по вертикали. Пинч-аут невозможен принципиально: при `width=device-width` layout-ширина равна экрану — уменьшать нечего. «Помельче» достижимо только layout viewport'ом шире экрана.

## Решения (брейншторм 2026-07-06)

1. **Мобилка — гибрид.** Портрет: доводим существующий колоночный режим (перенос топбара, естественный поток страницы вместо 100vh-клетки). Альбом на тач-устройствах: JS-свитч viewport-меты на `width=1180` → мини-десктоп, всё ×~0.78. Ровно то, что просил юзер («портрет разваливается, в альбоме не хватает масштабирования поменьше»).
2. **Состав спринта:** мобилка + UX-видимое + логика-минорки под TDD + a11y-пакет. **Инфра-спринт** (noUncheckedIndexedAccess, type-aware ESLint, CI на PR) — НЕ входит, остаётся в бэклоге отдельным решением.
3. **Витрина-виджет после reload** — минимум из бэклога: сказать про эфемерность (подпись у секции «Витрины» + строка в About). Глубокую регидрацию витрин не делаем.
4. **`updateWidgetTitle`** — полноценный click-to-edit заголовка виджета (зеркало подписи), не comment-only.

## Слайс 1 — мобилка

### Портрет (доводка `@media (max-width: 720px)`)

- **Топбар переносится:** `flex-wrap: wrap` в медиа-блоке (+ row-gap). Ожидаемый перенос: строка 1 — лого + `.mode-toggle`; строка 2 — `.topbar-right` (`?`, engine-tag, local, reset; правое выравнивание живо через `margin-left: auto`). Проверить глазами — юниформность строк, ничего не липнет.
- **Шелл в поток:** в колоночном режиме `.shell { height: auto; min-height: 100dvh }`, `.workspace`/`.body` не создают внутренних скролл-клеток — скроллится страница целиком, статуслайн в конце документа. Это убирает и «статуслайн посреди контента», и вертикальную давку.
- **Потолок виртуализованному гриду:** TanStack virtual требует ограниченный скролл-контейнер. В потоке страницы `.grid-scroll` (explore) теряет flex-ограничение → дать в медиа-блоке явный `max-height` (~60vh). В отчёте потолок 320px уже есть — не трогать.
- Горизонтальный оверфлоу после правки топбара перепроверить на всех экранах (welcome, explore с данными, отчёт): страница не должна скроллиться вбок.

### Альбом (тач-устройства)

- Новый модуль `src/mobileViewport.ts`:
  - **Чистая функция (TDD):** `viewportContentFor(orientation: 'portrait' | 'landscape', screenWidth: number): string` — landscape при экране меньше ~1100px → `'width=1180'`; иначе → `'width=device-width, initial-scale=1.0'`. Планшеты с широким экраном получают дефолт (не трогаем).
  - **Тонкая обвязка (глазами):** применяется только на тач-устройствах (`matchMedia('(pointer: coarse)')`); подписка на смену ориентации; своп `content` существующей меты `<meta name="viewport">`. Вызов из `main.tsx`. Десктоп не затронут (мета игнорируется).
- Поведение: телефон в альбоме видит цельную десктопную вёрстку в масштабе ~0.78, пинч-зум доступен; портрет возвращает `device-width` и колоночный режим.

## Слайс 3 — логика под TDD

Каждый пункт: красный тест → зелёный → рефактор.

1. **`dedupeColumnNames`:** `['id','id','id_1']` → второй `id_1` затирает первый; seen-set по уже выданным именам. XS.
2. **Guard `_qb_`-префикса имён файлов:** файл `_qb_raw_events.csv` клоберит внутреннюю raw-таблицу; `isInternalTable`-чек в `loadOneFile` → переименование с безопасным префиксом. XS.
3. **`dropMart` не глотает реальные ошибки:** на пойманной ошибке — тост (паритет с `refreshMart`); `removeDataset` всё равно выполняется (объект эфемерен). XS.
4. **Reserved-words в `validateMartName`:** denylist (`order`, `select`, `table`, …) — цель валидации «не кавычить в ручном SQL» должна выполняться. XS.
5. **In-flight guard двойного сабмита витрины:** disabled на время `createMart` (гонка `taken` → дубликат в сторе). XS.
6. **TOCTOU guard загрузки:** `loadDemoData` / `Shell.handleFiles` — in-flight promise/guard против двойного клика/дропа. XS.
7. **`exportQuery`:** уникальный суффикс имени виртуального файла + `try/finally`-очистка (двойной экспорт коллидирует; упавший `copyFileToBuffer` течёт). XS.
8. **`stripTrailingSemicolon` — консолидация:** один экспортируемый хелпер в `core/sql.ts` вместо трёх копий (`core/mart.ts`, инлайн в `buildResultTempDDL`, `duckdbClient.exportQuery`); заодно многократные `;` и хвостовые комментарии. XS–S.
9. **`cycleSort` в core:** asc→desc→remove + additive multi-sort из `ResultPanel` → `core/resultQuery.ts` под тесты (TDD-граница CLAUDE.md). S.
10. **`profileResult` — семантика «профиль отображаемого результата»:** paged-таб переиспользует снапшот последнего запуска (ок), raw-таб должен профилировать SQL последнего запуска, а не текущий черновик редактора. S.
11. **`loadedKey` + поколение материализации:** ключ из одних имён таблиц не видит re-apply схемы (`setApplied` пересоздаёт таблицу с тем же именем) → зависимая ячейка держит устаревший результат. Включить в ключ generation/seq материализации. S.
12. **Explore: двойной fetch окна:** view-эффект перезапрашивает страницу, уже лежащую в сторе (первый запуск, переключение таба). Guard «view реально поменялся». S.
13. **`extractDatasetNames` — сузить:** вычищать строковые литералы и комментарии, матчить идентификаторы только после `FROM`/`JOIN` (таблица `id` + `SELECT id FROM orders` больше не даёт ложный источник). S.
14. **`onRun={(sql) => commitDraft(sql)}`** в ячейке — выравнивание с explore-паттерном (сейчас аргумент CM игнорируется). XS.
15. **`Report.tsx` save():** отложенный `revokeObjectURL` (как в `downloadHtml`), синхронный может оборвать скачивание. XS.
16. **`pointercancel` на обоих ресайзах** (`SqlEditor`, rail): выход из драга при отмене/`ev.buttons === 0`. XS.
17. **Перерендер-минорки:** `React.memo(ResultGrid)`; мемоизация chart-spec (эффект `Chart` не перегоняет Plot на каждый рендер родителя). XS–S.
