# quackbook — Warp Redesign (spec, 2026-07-02)

> Пост-v1 визуальный редизайн: warp-drive WebGL-шейдер как hero-момент на входе + сдвиг айдентики в терминал-неон (циан/магента). Цель — передать **ощущение скорости/полёта** от утилиты. Firewall-clean: раскладка, структура данных и формат отчёта не меняются.

## Зачем

quackbook — технодемка для портфолио. Её продаёт ощущение, а не список фич. Сейчас входной опыт (прелоадер + welcome) функционален, но не производит впечатления. Warp-портал на входе + неон-акценты внутри дают эмоциональный крючок «быстро, живо, летит», не трогая рабочую суть тула.

## Решения (зафиксированы на брейншторме)

1. **Глубина — точечная.** Шейдер = hero на входе; остальное приложение сохраняет навигацию и раскладку, но акценты и типографика переезжают в терминал-неон. Не полный reskin-иммерсия.
2. **Дом шейдера — Boot + Welcome.** Boot эфемерен (виден раз на браузер до кэша WASM); Welcome виден каждую сессию — оба получают шейдер-фон.
3. **Палитра — cyberpunk циан/магента.** Янтарь уходит целиком.
4. **Технология — raw WebGL, 0 новых зависимостей.** three.js НЕ тянем. GLSL пишется в проекте (снимает и вопрос лицензии стороннего компонента).
5. **Цвет шейдера алайнится с палитрой** — кольца и RGB-split тонируются в циан↔магента, не референсная радуга.
6. **Движение внутри тула — спокойное ядро + быстрые акценты** (переходы 120–160ms, neon-sweep на focus/hover, короткий neon-pulse на «запустить»); всё под `prefers-reduced-motion → статика`.
7. **Ребрендинг: quackbook → warpbook** (отображаемый бренд). Лого — концентрические кольца-портал (как в шейдере), не утка.

## Вне скоупа (firewall)

- Раскладка приложения (вертикальный стек отчёта, ширина одна, табы, рейл) — не трогаем.
- Структура данных, стор, SQL-пайплайн, экспорт-логика — не трогаем.
- **Экспорт-тема отчёта остаётся светлой печатной** (`core/exportHtml.ts` `STYLE`) — редизайн касается только тёмного чрома приложения. Экспортируемый HTML/PDF не неонит.

- Иммерсивный шейдер-фон за всем приложением — отклонён (GPU/батарея во время анализа + риск контраста).

## Архитектура

### Новый компонент: `src/components/WarpShader.tsx`

Самодостаточный React-компонент: `<canvas>` + raw WebGL2 (fallback на WebGL1), GLSL-исходники в строковых константах. Единственная ответственность — рисовать warp-фон. Не знает про стор, данные, роутинг.

Пропсы:
```ts
interface WarpShaderProps {
  intensity?: number   // 0..1, множитель яркости/скорости (boot=1.0, welcome=0.85). default 1
  className?: string
}
```

Внутреннее устройство:
- `useEffect` на маунте: получить `gl`, скомпилировать программу (vertex = fullscreen quad, fragment = warp), залить геометрию (2 треугольника на весь клип), поднять RAF-цикл.
- Uniforms: `u_time` (сек), `u_resolution` (px, с учётом DPR), `u_mouse` (0..1, нормализованные, сглажены lerp), `u_intensity`.
- Cleanup: `cancelAnimationFrame`, удалить program/buffers, снять слушатели, потерять контекст (`getExtension('WEBGL_lose_context')?.loseContext()`).
- Мышь: слушатель `pointermove` на window (throttle через RAF-флаг), центр тоннеля слегка смещается за курсором (parallax); на тач/без движения — центр в середине.

### Потребители

`BootScreen` и `WelcomeScreen` рендерят `<WarpShader>` первым слоем (position:absolute, инсет 0, z-index 0), контент — вторым слоем (z-index 1) с полупрозрачной тёмной вуалью под текстом для контраста.

## Лого-глиф — `src/components/Icon.tsx`

Утиный глиф заменяется на **варп-портал**: 4–5 концентрических stroke-колец, центрированных, с уменьшающимся радиусом и небольшим сгущением к центру (эхо тоннеля шейдера). Двухтональность: внешние кольца — циан (`--accent`), внутренние сдвигаются к магенте (`--accent-2`) — либо чередование, либо плавный градиент по кольцам; крошечная яркая точка/колодец в центре. Опциональный лёгкий horizontal-offset между цветовыми кольцами для намёка на RGB-split (но читаемость на 18–22px — приоритет, stroke ~1.5–1.8). Ветка `logo` в `Icon.tsx` рисует кольца (остальные глифы — прежние stroke-иконки). Комментарий про утку/янтарь заменить.

## Warp-шейдер — GLSL

Полноэкранный fragment. Псевдо-логика:
- `uv` = нормализованные координаты, центр = `0.5 + (u_mouse-0.5)*parallax`.
- `p = uv - center`; `r = length(p)`; `a = atan(p.y, p.x)`.
- Кольца-тоннель: `rings = sin(log(r) * FREQ - u_time * SPEED)`; чем ближе к центру — тем плотнее (эффект ускорения к порталу).
- Свечение колец: узкие яркие полосы через `pow(smoothstep(...), k)`.
- **Хроматическая аберрация в палитре:** канал 1 сэмплируется на `r + offset` и красится в **магенту** (`#e849c4`), канал 2 на `r - offset` в **циан** (`#22d3ee`), середина — их смесь; `offset` растёт к краям (усиление split на периферии, как в референсе).
- Виньетка к краям (тёмный `--bg`), центр — тёмный колодец, из которого «вылетают» кольца.
- `u_intensity` множит яркость и `SPEED`.

Ноль текстур, ноль зависимостей. Один проход.

## Гварды производительности и доступности

- **DPR-кап:** рендер-буфер = `min(window.devicePixelRatio, 2) * cssSize` — не палим 4K-fragment на ретине.
- **Пауза:** RAF останавливается на `document.hidden` (`visibilitychange`) и при анмаунте компонента. Скрытая вкладка / уход с welcome не жгут GPU.
- **`prefers-reduced-motion: reduce`:** RAF не поднимается вообще — рисуем ОДИН статичный кадр (`u_time=0`), портал застывший, но красивый. Проверяем через `matchMedia`.
- **Фоллбэк без WebGL / context lost:** если `getContext('webgl2'||'webgl')` вернул null или контекст потерян — компонент рендерит статичный CSS-фон (radial-gradient циан→магента→bg) вместо canvas. Никакого белого/пустого экрана. Обработчик `webglcontextlost` → перейти на CSS-фоллбэк.

## Boot / Welcome — интеграция

**BootScreen:** `<WarpShader intensity={1}>` фоном; существующая карточка (лого + честный `%` + indeterminate-бар из M8) — поверх, на полупрозрачной подложке `--surface` с blur. Прогресс-логика (`bootPercent`/`formatMb`) **не трогается**.

**WelcomeScreen:** `<WarpShader intensity={0.85}>` фоном; карточка онбординга (заголовок + 3 шага + 2 CTA + кредит) — поверх, с вуалью для контраста. Логика загрузки демо/отчёта **не трогается**. CTA-кнопки получают neon-стиль (см. ниже).

## Акценты и типографика

**Неон получают (element-by-element):**
- Primary-кнопки «запустить» / «закрепить» / welcome-CTA: фон `--grad-neon` или циан-заливка + `--glow-cyan`; hover — усиление glow.
- `:focus-visible` по интерактиву: `--glow-cyan` ring (быстрый sweep-переход).
- Активный таб (`.tab.active` / mode-toggle `.on`): неон-андерлайн/бордер циан + лёгкий glow.
- Подсветка рейла (активный датасет, `.rail` highlight), ссылки (`a`): циан; hover — магента-сдвиг.
- «Опасные»/reset — `--danger` (неон-красный).

**Типографика (терминал-вайб):**
- JetBrains Mono шире: топбар-лого-текст `> quackbook`, лейблы секций рейла, текст кнопок-тулбара, статусные строки. Курсор-глиф `_` у лого (моно) — опционально, статичный (мигание только вне reduced-motion).
- Inter остаётся для тела markdown-текста в отчёте и welcome-lead (читаемость длинного текста).

**Чарты (dataviz):** серии Observable Plot переводим на неон-палитру (циан primary, магента secondary, далее — производные с достаточным контрастом на тёмном). Держим читаемость: не радуга, 2–4 различимых hue. Правки в `Chart.tsx`/`plotFigure.ts` (цвет линии/бара) и, если есть, в токенах серий. Экспорт-тема (светлая) — свои цвета, не трогаем.

**SQL-подсветка (`SqlEditor.tsx`):** текущая amber-on-teal `HighlightStyle` (веха M6) ретюнится под циан/магента (ключевые слова — циан, строки/числа — тёплый `--warn` или магента, идентификаторы — `--text`). Читаемость и контраст — приоритет.

## Движение

- Глобально: `transition: … 140ms ease-out` на интерактивных состояниях (hover/focus/active) — там, где сейчас нет или дефолт.
- neon-sweep: на focus/hover kнопок/табов — быстрый прогон glow (псевдоэлемент или box-shadow transition).
- «запустить» → короткий neon-pulse (150–250ms) на кнопке/панели результата при старте запроса (CSS-класс на время in-flight; вешается на существующий loading-флаг, без новой логики стора).
- `@media (prefers-reduced-motion: reduce)`: все добавленные анимации/переходы → мгновенные/статичные; шейдер застывает (см. гварды).

## Стратегия тестирования

Чистая презентация (CSS/GLSL/визуальные компоненты) — **проверяется глазами** (граница TDD из CLAUDE.md), не под юнит-тестами. Существующие **285 тестов должны остаться зелёными** (веха не трогает логику) — это регрессионный гейт.

Под TDD — только если появится чистая тестируемая логика. Кандидаты:
- `hexToGlColor(hex): [number,number,number]` (если понадобится передавать палитру в шейдер из JS, а не хардкодить в GLSL) — маленькая чистая функция, TDD.
- `shouldAnimateWarp({ reducedMotion, hidden }): boolean` — если решим вынести решение о RAF в чистый хелпер, TDD.
Если обе останутся тривиально-инлайн в компоненте — тестами не покрываем (презентация).

Гейт каждой задачи: `npm test` (285+ зелёных) + `npm run build` (полный type-check) + `npm run lint` (0/0). Визуальная приёмка — пользователем в `npm run dev` (шейдер/палитра/движение агенты в node не проверяют).

## Риски

- **Контраст текста над шейдером** — welcome/boot: обязательна вуаль/подложка под контентом, иначе читаемость страдает. Проверяется глазами.
