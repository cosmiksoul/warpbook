# M7b «Исполняемые ячейки» — дизайн

Дата: 2026-07-05. Статус: принят (брейншторм в сессии, решения пользователя зафиксированы ниже). Контекст: второй милстоун пост-v1 очереди `docs/ROADMAP.md`; историческое имя — из декомпозиции M7 на M7a «Витрины» + M7b.

## Цель

Отчёт = живой ноутбук: SQL виджета правится и исполняется на месте, «выполнить всё» пересчитывает документ, новый запрос рождается прямо в отчёте.

## Решения пользователя (2026-07-05)

1. **Inline-редактор** CodeMirror прямо в ячейке (правка = черновик, «▸ выполнить» коммитит); НЕ roundtrip в explore.
2. **Статусы**: `выполняется` / `ошибка` / `правка не выполнена`; свежий ок — без чипа. Смена данных пересчитывает автоматически (существующий `loadedKey`-механизм), отдельного «устарел по данным» нет.
3. **«Выполнить всё»**: сверху вниз; ошибка ячейки не прерывает остальные.
4. **«+ запрос»** в тулбаре отчёта.
5. **Формат отчёта не меняется** (version 1; статусы — runtime-only). Старых отчётов в природе нет (проект без пользователей), совместимость — не ограничение, но и не ломаем зря.

## Модель

- Типы `Block`/`WidgetBlock` (`core/report.ts`) не меняются. Пустой `sql` — легальное состояние нового query-блока: не исполняется, ячейка показывает хинт «напиши запрос и нажми ▸».
- **Черновик** — локальный стейт `WidgetBlockView` (не сериализуется). `draft !== block.sql` → чип «правка не выполнена». Черновик живёт, пока блок смонтирован: переключение в explore и обратно сбрасывает невыполненную правку — принято (runtime-only, как статусы).
- **Коммит** — store-экшен `updateWidgetSql(id, sql, datasetNames)`: пишет оба поля; автосейв отчёта в localStorage срабатывает штатно; существующий rerun-эффект (`[block.sql, client, loadedKey]`) пересчитывает ячейку.
- **Пере-исполнение без правки** («⟳») — локальный `runSeq`, добавленный в депсы эффекта.
- `datasetNames` пере-выводятся на момент коммита: `extractDatasetNames(sql, knownTables)` по известным таблицам стора — rehydration-подсказка не протухает.

## UI — `WidgetBlockView`

- Тоггл «SQL ▸» открывает **редактор вместо `<pre>`**: переиспользуем `SqlEditor` (автокомплит по схеме, та же подсветка) в компактном режиме (~120 px, без drag-resize); если сцепление с explore-режимом окажется дорогим — тонкий `CellSqlEditor` на тех же CM6-экстеншенах (решение за планом, поведение одинаковое). Ctrl+Enter = выполнить.
- Кнопки строки редактора: **«▸ выполнить»** (disabled при пустом черновике), **«⟳»** (пере-исполнить сохранённый sql), **«отменить»** (draft ← block.sql).
- Чип статуса в `widget-head`: `…` (выполняется — существующий loading), `ошибка` (существующий error-блок остаётся ниже), `правка не выполнена` (акцент-2/dim). Свежий ок — чипа нет.
- Новый query-блок (`sql === ''`): редактор раскрыт сразу, эффект скипает пустой sql (новый kind `idle` вместо вечного «пересчитываю…»).

## Тулбар отчёта

- Левая группа: «+ текст» · «+ код» · **«+ запрос»** — вставляет widget-блок `{title: 'запрос', sql: '', datasetNames: [], vizType: 'table', caption: ''}` с той же семантикой места вставки, что у «+ текст» (план сверяет фактическую семантику вставки/активного блока и переиспользует её).
- Правая группа: **«▸ выполнить всё»** перед экспорт-группой. Механика: store-счётчик `runAllSeq`, инкремент по кнопке; каждый widget-эффект получает `runAllSeq` в депсы → все ячейки диспатчатся в порядке документа, единственный WASM-worker сериализует исполнение → наблюдаемо последовательно сверху вниз; пустые sql скипаются; ошибки локальны для ячейки.

## Store — `state/session.ts`

- `updateWidgetSql(id, sql, datasetNames)` — редьюсер (TDD по паттерну существующих стор-тестов).
- `runAllSeq: number` + `runAll()` — инкремент (TDD).
- Вставка query-блока — переиспользование/минимальное расширение существующего пути вставки блоков (как `addTextBlock`).

## Тестирование и гейт

- TDD: `extractDatasetNames`; редьюсеры `updateWidgetSql` / `runAll`; guard пустого sql (если вынесен в чистую функцию — тестом, иначе глазами).
- Путь исполнения ячейки не меняется (`client.query(buildWidgetSql(...))` тот же) — новых node-интеграций с движком не требуется. Черновик/чипы/«выполнить всё» — глазами.
- Гейт: `npm test` + `npm run build` + `npm run lint` 0/0; **0 новых npm-зависимостей** (CodeMirror уже в проекте).
