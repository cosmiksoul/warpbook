## Открытые

- [ ] **Dev-консоль: sourcemap-шум DuckDB-WASM.** Vite dev-сервер печатает пачку «Sourcemap for …/duckdb-browser-eh.worker.js points to a source file outside its package» (воркеры `@duckdb/duckdb-wasm` ссылаются на исходники `@duckdb/apache-arrow/*.ts` вне пакета). Косметика, **только в `npm run dev`** — в `build`/на Pages этого нет; не наш код. Фикс: `customLogger` в `vite.config.ts`, глушащий ровно эти строки (~10 строк). Оценка: XS. Источник: сессия 2026-06-23 (скрин dev-консоли).

- [ ] **Семантика «сброс» vs «сбросить».** Хедерная **«сброс»** возвращает всю схему к исходным VARCHAR (раз-типизировать всё); в редакторе колонки **«сбросить»** возвращает колонку к *инференс-предложению*. Два разных «reset». Решить: унифицировать (оба → VARCHAR, либо оба → suggested) или переименовать для ясности (напр. хедер «сбросить типы», в колонке «к предложенному»). Оценка: S. Источник: сессия 2026-06-23 (UX-фидбек по M2).

- [ ] **Метка «применить к колонке».** Кнопка в редакторе колонки ре-материализует **всю** таблицу (из текущего конфига), а не только эту колонку. Сейчас метка ок по смыслу «применить правку колонки», но если будет путать — переименовать (напр. «применить»). Оценка: XS. Источник: сессия 2026-06-23.

- [ ] **Неотслеживаемые фикстуры.** В рабочем дереве висят `fixtures/acceptance.sql` и `fixtures/events.csv` (M1-овые приёмочные артефакты). Решить: закоммитить в `fixtures/`, добавить в `.gitignore`, или удалить. `fixtures/dirty.csv` уже закоммичен (M2). Оценка: XS. Источник: сессия 2026-06-23.
