# Backlog — отложенный полишинг

Мелкие, **неблокирующие** правки, которые копим и закрываем **одним скоупом в полишинг-спринт**. Сознательно не тащим их поштучно в текущие вехи — это держит фокус и хирургичность правок (CLAUDE.md rule 2/3).

**Статус: M5 «Экспорт и полировка» прошёл (2026-06-26)** — полишинг-спринт закрыл почти весь накопленный список (см. «Сделано»). Открытым осталось то, что план M5 намеренно отложил, плюс пара известных-принятых миноров из финального ревью M5.

Формат: `[ ]` открыто / `[x]` сделано. Поля: область · что · почему отложено · оценка · откуда.

### Известные-принятые (минор, из финального review M7a «Витрины» — не баги, фиксируем чтоб не потерять)

- [ ] **`dropMart` глотает реальные ошибки молча.** `useMartActions.ts` — пустой `catch` оправдан «DROP IF EXISTS идемпотентен», но `IF EXISTS` гасит только «не существует»; ошибка зависимости (удалить витрину, на которую ссылается VIEW другой витрины — путь, который фича сама открывает) бросается, глотается, а `removeDataset` всё равно срабатывает → витрина исчезает из рейла без тоста, объект остаётся в каталоге DuckDB. Несогласованность с `refreshMart` (тот тостит). Фикс (top-кандидат): на пойманной ошибке — тост (как в `refreshMart`), объект всё равно эфемерен → на reload уйдёт. Оценка: XS. Источник: M7a финальный review (code-quality lens).
- [ ] **Reserved-word имена проходят валидацию.** `validateMartName` пропускает `order`/`select`/`table` (совпадают с `^[A-Za-z_][A-Za-z0-9_]*$`); в DDL работают через `quoteIdent`, но заявленная цель валидации («чтоб не нужно было кавычить в ручном SQL/автокомплите») нарушается — `SELECT * FROM order` в другом табе = syntax error. Редкий foot-gun; закрыть denylist'ом при желании. Оценка: XS. Источник: M7a финальный review.
- [ ] **Гонка двойного сабмита → дубликат в сторе.** `createMart` читает `taken` синхронно, потом ждёт `exec`+`describeTable` до `addDataset`; зажатый Enter (key-repeat) может сложить несколько вызовов, все проходят коллизию против устаревшего `taken` → два `<li>` с одним key (React-варнинг). Само-лечится (`dropMart` фильтрует все совпадения), одиночный юзер, низкая вероятность. Фикс: disabled/in-flight guard на кнопках сабмита. Оценка: XS. Источник: M7a финальный review.
- [ ] **Витрина-виджет в отчёте после перезагрузки — вечная ошибка + вводящий в заблуждение хинт.** Пин виджета на `SELECT … FROM my_mart` пишет `my_mart` в `datasetNames`; отчёт персистится в localStorage, витрины — нет. После reload виджет перезапускается и падает навсегда с хинтом «источник(и): my_mart — подгрузи», хотя витрину нельзя подгрузить файлом. Спека осознанно отложила прямой пин витрин и регидрацию — принятое следствие, но острый край. Минимум: где-то сказать юзеру «витрины живут до перезагрузки» (тултип витрины / About). Оценка: S. Источник: M7a финальный review (firewall/spec lens).
- [ ] **Пост-Reset утечка каталога → возможная коллизия при загрузке файла.** `reset()` чистит стор, но VIEW/TABLE объекты остаются в каталоге DuckDB (`Shell.handleReset` шлёт `DROP TABLE IF EXISTS` даже для view → ошибка, глотается). Если без reload загрузить файл с именем, равным осевшей витрине, `uniqueTableName` (по стору, теперь пустому) клэш не заметит, а `CREATE OR REPLACE TABLE` поверх осевшего VIEW бросит → «Не удалось загрузить». Узко (reset + то же имя + без reload); спека отложила чистку каталога. Оценка: S. Источник: M7a финальный review.
- [ ] **`refreshMart` переставляет витрину в конец списка + `stripTrailingSemicolon` в третьей копии.** Косметика: `removeDataset`+`addDataset` в refresh аппендит → снапшот прыгает вниз «Витрин» (in-place update убрал бы). И приватный `stripTrailingSemicolon` (`core/mart.ts`) — третья копия того же `sql.trim().replace(/;\s*$/,'').trim()` (уже инлайн в `sql.ts` `buildResultTempDDL` и `duckdbClient.ts` `exportQuery`); один экспортируемый хелпер в `sql.ts` консолидировал бы все три. Оценка: XS. Источник: M7a финальный review.

### Известные-принятые (минор, из финального review M5 — не баги, фиксируем чтоб не потерять)
